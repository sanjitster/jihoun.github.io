/dmo

A dmo is a generic program. It holds all the typical informations of a program. Such as, its name, its picture, its unit... The link between a dmo and a user is a card.

/dmo/all/{date} get

get

Get the list of dmos updated since a specific date. Be careful, this will only return active dmos. Meaning if you fetched the definition of a dmo active in the past, but it is inactive now, you will not receive the update. For this, you should use the find method.

get /dmo/all/{date}

Get the list of dmos updated since a specific date. Be careful, this will only return active dmos. Meaning if you fetched the definition of a dmo active in the past, but it is inactive now, you will not receive the update. For this, you should use the find method.

  • Request
  • Response

URI Parameters

  • date: (integer )

    timestamp up to when the dmo should not be fetched.

HTTP status code 200

Body

Type: application/json

/categories

All dmos can belong to one or more categories. Categories canbe things like Fashion and Beauty, Travel and Hotel, ...

/categories/all/{date} get

get

Get the full list of available categories.

get /categories/all/{date}

Get the full list of available categories.

  • Request
  • Response

URI Parameters

  • date: (integer )

    Timestamp. If specified, only categories updated after this timestamp will be returned.
