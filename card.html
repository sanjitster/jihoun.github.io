/card

A card is the link between a program and a user. The card holds a value, a card number, an eventual password, etc.

/card/all/{date} get

get

Get the list of cards accessible by the current user.

  • Response

/card/credit post

post

Credit a Loyalty program. To credit a gift card, use the cart api.

post /card/credit

Credit a Loyalty program. To credit a gift card, use the cart api.

Secured by oauth_2_0
  • Request
  • Response

Body

Type: application/x-www-form-urlencoded

Form Parameters
  • card_number: required (string )
  • amt: required (number )
  • exchange_id: (string )
  • type: (one of cash,points- default: points )

    If "cash", it means that amt is specified in cash. In that case a valid exchange_id needs to be specified. Otherwise, if "points" it means that the amount is specified in the card unit.

/transaction

Access to transactions on a card. For giift issued cards, the value of the card is equal to the sum of transactions values.

  • For a transaction removing value from a card (debit), the sign will be negative.
  • For transactions crediting a card, the sign will be positive.
A user only has access to transactions on his cards.
