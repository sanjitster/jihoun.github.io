post /buy/card

Allows buying one or multiple gift cards in a single api call. This is only available to retailers with previous agreement. To have access to the different steps of the process the cart, order, payment and delivery apis should be used.

Secured by oauth_2_0
  • Request
  • Response

Body

Type: application/x-www-form-urlencoded

Form Parameters
  • dmoId: required (string )

    Id of the program to be bought.

  • amount: required (number )

    Face value of the gift card to be bought.

    Example:

    25
  • units: (integer- default: 1 )

    Number of gift cards to be bought.

  • state: (string )

    Optional external id that will be visible in the response.

    Example:

    EXTERNAL_ID_123213423
  • numbers: (string )

    List of numbers that should be assigned to the cards being issued. This feature is not available for all programs (dmos). And some numbers might already be taken. In that case an error is returned. It is usually better to omit this parameter. Numbers should be separated by commas.

    Example:

    4444333322221111,4444333322221112
  • delivery_type: (one of api,offer- default: api )

    Defines if this card is delivered directly in the response of this api call. Or if an email should be sent to someone with the card details.

  • delivery_data_rname: (string )

    Name of the person receiving the card. Recommended if delivery_type=='offer'

    Example:

    John Connor
  • delivery_data_remail: (string )

    Email of the person receiving the card. Mandatory if delivery_type=='offer'

    Example:

    john.connor@skynet.com
  • delivery_data_msg: (string )

    Email of the person receiving the card. Mandatory if delivery_type=='offer'. Useless otherwise.

    Example:

    Happy Go Live
  • delivery_data_sname: (string )

    Name of the person sending the card. Recommended if delivery_type=='offer'. Useless otherwise.

    Example:

    Sarah
  • delivery_data_semail: (string )

    Email of the person sending the card. Recommended if delivery_type=='offer'. Useless otherwise.

    Example:

    sarah@skynet.com
  • delivery_data_date: (date )

    Date at which the card should be delivered. Default is today. Optionnal if delivery_type=='offer'. Useless otherwise. Format is YYYY-MM-DD.

    Example:

    2011-04-11

Type: application/json

Example:

{
  "dmoId":"1459820042qpC87",
  "amount":42,
  "units":1,
  "state":"EXTERNAL_ID_123213423",
  "numbers":["4444333322221111"],
  "delivery":{
    "type":"offer",
    "data":{
      "r_name":"John Connor",
      "r_email":"john.connor@skynet.com",
      "msg":"Happy Go Live",
      "date":"2011-04-11",
      "from_name":"Sarah",
      "from_mail":"sarah@skynet.com"
    }
  }
}
